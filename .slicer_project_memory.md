# SlicerClaw Project Evolution Memory

This file is managed by your AI assistance to store lessons, codebase context, and API gotchas exclusive to the `slicerClaw` development project itself. It is designed to be version-controlled (git) so multiple developers and AI agents can sync the shared project knowledge.

## 🧠 Project Architecture Insights
- The `SlicerClaw` backend core is implemented as a 3D Slicer Scripted Loadable Module in `SlicerClaw.py`.
- MCP (Model Context Protocol) is provided via an embedded python TCP server inside Slicer, running by default on `127.0.0.1:2016`.
- The user interacts with the extension via Slicer UI (PythonQt built programmatically) and an OS-level spotlight bar hook (invoked via Cmd/Ctrl+I).

## ⚠️ Known Implementation Gotchas (Slicer API)
- **UI Element Deletion:** When reloading the module or cleaning up during developer tests, use `deleteLater()` on QWidgets rather than just dereferencing them. We encountered issues where `slicer.ai_spotlight_chat.hide()` works but garbage collection doesn't remove the hook cleanly unless explicitly destroyed.
- **Mac Frameless Window Activation**: The spotlight UI window floats without default OS frames. On macOS (`slicer.app.os == 'darwin'`), simply calling `.show()` might not fully seize the keyboard focus. We use `self.raise_()`, `self.activateWindow()`, and explicitly `self.input_edit.setFocus()` to ensure text inputs function properly.
